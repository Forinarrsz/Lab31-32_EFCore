## 1. Описание работы
Данный проект представляет собой REST API для управления списком задач. В работе реализовано взаимодействие с базой данных SQLite с использованием подхода **Code First** и Entity Framework Core. API поддерживает фильтрацию, поиск, пагинацию и массовые операции.

## 2. Полезные команды dotnet ef
* `dotnet ef migrations add Name` — создание новой миграции.
* `dotnet ef database update` — применение миграций к базе данных.
* `dotnet ef migrations remove` — удаление последней миграции.

## 4. Реализованные маршруты (API)
* `GET /api/tasks` — получение всех задач (с фильтрами).
* `GET /api/tasks/search` — поиск по тексту.
* `GET /api/tasks/stats` — статистика выполнения.
* `GET /api/tasks/paged` — пагинация.
* `GET /api/tasks/overdue` — список просроченных задач.
* `POST /api/tasks` — создание задачи.
* `PUT /api/tasks/{id}` — полное обновление.
* `PATCH /api/tasks/{id}/complete` — переключение статуса.
* `PATCH /api/tasks/complete-all` — массовое завершение.
* `DELETE /api/tasks/completed` — удаление выполненных.

## 5. Сравнительная таблица


| Концепция | Хранение в памяти (static List) | EF Core + SQLite |
| :--- | :--- | :--- |
| **Хранение данных** | static List<T> в RAM | Файл .db на диске |
| **После перезапуска** | Данные пропадают | Данные сохраняются |
| **Поиск по условию** | LINQ to Objects | LINQ to Entities → SQL |
| **Создание структуры** | Не нужно | Миграции (dotnet ef) |
| **Начальные данные** | Хардкод в коде | HasData() в миграции |
| **Получение данных** | list.FirstOrDefault(...) | await db.Table.FindAsync(id) |
| **Масштабируемость** | Ограничена RAM | Гигабайты данных |
| **Транзакции** | Нет | Встроены в EF Core |

## 6. Главные выводы
1. **EF Core** — это переводчик между C# и SQL. Мы пишем LINQ, он генерирует SQL.
2. **Миграции** — система контроля версий для структуры БД (как Git для кода).
3. **Code First** удобнее ручного SQL: изменил класс → создал миграцию → база обновлена.
4. **SaveChangesAsync()** — ключевой момент. До него изменения живут только в памяти.
5. **Async/await** при работе с БД — это стандарт. Нельзя блокировать поток во время ожидания ответа от диска/базы.
